利用CocoaPods打包静态库

CocoaPods作为OS X和iOS开发平台的类为管理工具，目前非常完善。通常我们用pod来管理第三方开源类库，同时在公司内部也会利用pod管理依赖关系的静态库给其它人使用，以及开发相关SDK(不愿意公开源代码),那么就需要打包成.a文件。这次的GDLib Demo是以一个依赖于AFNetworking的静态类库。

关于CocoaPods的资料参考:cocoapods官方文档

开发静态库(Static Library)

创建静态库有两种方式。这里介绍基于pod自动创建，以GDLib为项目演示。

1.在终端执行命令pod lib create GDlib。


这样会创建一个GDLib的项目,GDLib文件二级目录结构:

2.打开GDLib.xcworkspace文件。

配置信息如下，可根据需求进行修改。

#

# valid spec before submitting.
#
# Any lines starting with a # are optional, but their use is encouraged
# To learn more about a Podspec see http://guides.cocoapods.org/syntax/podspec.html
#

Pod::Spec.new do |s|
  s.name             = 'GDLib'    #名称
  s.version          = '0.1.0'    #版本号
  s.summary          = 'A short description of GDLib.'    #简短介绍

# This description is used to generate tags and improve search results.
#   * Think: What does it do? Why did you write it? What is the focus?
#   * Try to keep it short, snappy and to the point.
#   * Write the description between the DESC delimiters below.
#   * Finally, don't worry about the indent, CocoaPods strips it!

  s.description      = <<-DESC
TODO: Add long description of the pod here.
                       DESC

  s.homepage         = 'https://github.com/<GITHUB_USERNAME>/GDLib'                            #主页地址
  # s.screenshots     = 'www.example.com/screenshots_1', 'www.example.com/screenshots_2'    #截图
  s.license          = { :type => 'MIT', :file => 'LICENSE' }    #开源协议
  s.author           = { 'geeven' => '598800360@qq.com' }        #作者信息
  s.source           = { :git => 'https://github.com/<GITHUB_USERNAME>/GDLib.git', :tag => s.version.to_s }        #项目地址
  # s.social_media_url = 'https://twitter.com/<TWITTER_USERNAME>'                            #多媒体链接及介绍

  s.ios.deployment_target = '8.0'        #支持平台版本号

  s.source_files = 'GDLib/Classes/**/*' #源码文件

  # s.resource_bundles = {

添加本地代码库

1.修改s.source，修改路径如下

s.source = { :git => '/Users/geeven/Desktop/GDLib', :tag => '0.1.0' }

2.提交源码，并打tag

git add .
git commit -a -m 'v0.1.0'
git tag -a 0.1.0 -m 'v0.1.0'

验证类库

开发完成静态类库之后，需要运行pod lib lint验证一下类库是否符合pod的要求。
执行命令pod lib lint
出现错误根据提示再次执行代码pod lib lint --allow-warnings --no-clean

打包类库

需要使用一个cocoapods的插件cocoapods-packager来完成类库的打包。当然也可以手动编译打包，但是过程会相当繁琐。

  • 安装打包插件
    执行命令sudo gem install cocoapods-packager
  • 打包
    执行命令pod package GDLib --library --force
    其中–library指定打包成.a文件，如果不带上将会打包成.framework文件。–force是指强制覆盖。最终的目录结构如下
