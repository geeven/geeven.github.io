利用CocoaPods打包静态库

发表于 2016-09-08

CocoaPods作为OS X和iOS开发平台的类为管理工具，目前非常完善。通常我们用pod来管理第三方开源类库，同时在公司内部也会利用pod管理依赖关系的静态库给其它人使用，以及开发相关SDK(不愿意公开源代码),那么就需要打包成.a文件。这次的GDLib Demo是以一个依赖于AFNetworking的静态类库。

关于CocoaPods的资料参考:

cocoapods官方文档

开发静态库(Static Library)

创建静态库有两种方式。这里介绍基于pod自动创建，以GDLib为项目演示。

1.在终端执行命令pod lib create BZlib。


这样会创建一个GDLib的项目,GDLib文件二级目录结构:

添加本地代码库

1.修改s.source，修改路径如下

s.source = { :git => '/Users/geeven/Desktop/GDLib', :tag => '0.1.0' }

2.提交源码，并打tag

git add .
git commit -a -m 'v0.1.0'
git tag -a 0.1.0 -m 'v0.1.0'

打包类库

需要使用一个cocoapods的插件cocoapods-packager来完成类库的打包。当然也可以手动编译打包，但是过程会相当繁琐。

  • 安装打包插件
    执行命令sudo gem install cocoapods-packager
  • 打包
    执行命令pod package GDLib --library --force
    其中–library指定打包成.a文件，如果不带上将会打包成.framework文件。–force是指强制覆盖。最终的目录结构如下
