Cocoapods

在iOS开发中，我们经常利用Cocoapods来管理相关库，利用gem来安装Cocoapods。

因部分源因需要taobao镜像更改为oschina镜像:RubyGems 镜像- Ruby China

利用oschina镜像来安装

{
https://ruby.taobao.org/    #已不再维护
//  删除源
      $ sudo gem sources --remove https://rubygems.org/
//  添加源
    $ sudo gem sources -a     https://gems.ruby-china.org/ 
//  查看gem源    
    $ gem sources -l
    *** CURRENT SOURCES ***
    https://ruby.taobao.org/ 
//  安装
    $ sudo gem install cocoapods
    //  设置
    $ pod setup
}

常用命令

{
//  查看版本
    $ gem --version
//  创建默认的 Podfile
    $ pod init
//  第一次使用安装框架
    $ pod install
//  安装框架，不更新本地索引，速度快
    $ pod install --no-repo-update
//  今后升级、添加、删除框架，或者框架不好用都使用这个命令
    $ pod update
//  更新框架，不更新本地索引，速度快
    $ pod update --no-repo-update
//  搜索框架
    $ pod search XXX    
//  帮助
    $ pod --help
}

当执行$ pod search XXX时发现显示的tag版本低于最新版时，需要升级Cocoapods，升级过程和重新安装基本一样

安装出错解决

当执行如下运行时出错时

$ sudo gem install cocoapods

ERROR:  While executing gem ... (Errno::EPERM)
Operation not permitted - /usr/bin/xcodeproj

用如下命令即可解决

$ sudo gem install -n /usr/local/bin/ cocoapods --pre

gem服务器在国外，原本安装方式但需要翻墙(不建议使用,太耗时)

{
//  查看gem源
    $ gem sources –l
//  gem自身升级
    $ sudo gem update --system
//  查看版本
    $ gem --version
//  清除过期的gem
    $ sudo gem cleanup
//  安装包
    $ sudo gem install cocoapods
//  删除包
    $ gem uninstall cocoapods
//  更新包
    $ sudo gem update
//  列出本地安装的包
    $ gem list
}
