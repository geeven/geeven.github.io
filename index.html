mac本地服务器配置

发表于 2016-06-07  |  2016-09-22   |  

在开发过程中，为了测试方便而又无服务器的情况下，可以利用mac本地配置服务器临时测试。

修改配置文件

cd /etc/apache2/              进入 apache2 所在的文件目录;
ls                          查看当前文件夹下的文件目录.
sudo vim httpd.conf         sudo 以管理员的权限打开. vim
/DocumentRoot               查找DocumentRoot (Web服务器的)根路径
shift + v                     选中这一行
y                              复制这一行
p                              在当前行下面粘贴复制那一行
此处需要复制两句，并将当前文本路径更变
/Options Follow             查找这一行
在 Options Follow 中间插入 Indexes  让服务器以列表的形式呈现内容.

配置完毕之后的样子 Options Indexes FollowSymLinks Multiviews

/php  查找php 所在的那一行.
删除行首的 # 号.
esc  退出编辑模式.
:wq  保存并退出 vim 编辑器.

开启及关闭服务器

  • 启动服务器
    sudo apachectl -k restart or sudo apachectl restart

  • 关闭服务器
    sudo apachectl -k stop or sudo apachectl stop

注：如果配置服务器过程中出现以下提示,说明服务器配置成功:

AH00558: httpd: Could not reliably determine the server's fully qualified domain name, using teacherdeiMac.local. Set the 'ServerName' directive globally to suppress this message

httpd not running, trying to start

加密之RSA

发表于 2016-06-07  |  2016-09-22   |  

前段时间在app中加入了JSPatch以方便快速修复线上bug，在设计中考虑安全性，就使用了相关加密算法，其中包括RSA。本文就针对RSA进行了部分总结。

基础知识

  • 什么是RSA？
    RSA算法是第一个能同时用于加密和数字签名的算法,它是一种非对称加密算法。
  • 加密和加签有什么区别？
    • 加密：公钥放在客户端，并使用公钥对数据进行加密，服务端拿到数据后用私钥进行解密；
    • 私钥放在客户端，并使用私钥对数据进行加签，服务端拿到数据后用公钥进行验签。
    • 前者完全为了加密；后者主要是为了防恶意攻击，防止别人模拟我们的客户端对我们的服务器进行攻击，导致服务器瘫痪。

简介

苹果在 iOS 7.0.3 版本以后公布钥匙串访问的SDK,钥匙串访问接口是纯C语言的, 密码可以直接使用明文,钥匙串访问本身是使用 AES256加密,就是安全的.所以使用的时候,直接传递密码明文就可以了.
钥匙串访问的第三方框架(SSKeychain),是对 C语言框架 的封装.
钥匙串访问的密码保存在哪里只有苹果才知道.这样进一步保障了用户的密码安全.

使用步骤:

  1. 导入第三方框架SSkeychain
  2. 利用钥匙串存储密码，将用户密码保存在钥匙串中
  1. 利用钥匙串取密码

常用Git命令

发表于 2015-06-07  |  2016-09-22   |  

一般来说，日常使用只要记住6到7个命令，就可以了。但是熟练使用，恐怕要记住很多个命令。

常用命令

  • git clone path:将远程服务器代码clone到本地path路径上
  • git status :查文件的状态
    • 查看某个文件的状态:git status 文件名
    • 查看当前路径所有文件的状态:git status

  • git init:初始化一个空的本地仓库，生成一个.git目录，用于维护版本信息

    • 在当前路径初始化仓库:git init
    • 在其他路径初始化仓库:git init 仓库路径

  • git add : 将工作区的文件保存到暂缓区
    保存某个文件到暂缓区: git add 文件名
    保存当前路径的所有文件到暂缓区: git add .

  • git commit :将暂缓区的文件提交到当前分支
    提交某个文件到分支:git commit -m ”注释” 文件名
    保存当前路径的所有文件到分支:git commit -m ”注释”

  • git pull:相当于是从远程获取最新版本并merge到本地

  • git fetch path:相当于是从远程获取最新版本到本地，不会自动merge
  • git push:将本地分支推送到远程服务器，保持远程与本地一致
